FAQ

What are these starters I keep seeing?

Each starter represents an empty template to begin with, having the components in the name present.

 

How do I install the item?

Copy and paste all the items in the code folder to your server to your desired location. 

 

What are those i class"fa " things?

That is Font Awesome, the amazing icons we used for this item you can learn more about them and find a cheatsheet on how to use them here http://fortawesome.github.io/Font-Awesome/icons/

 

How do I change the contact form information?

There is only one PHP file in the item, please open that item and replace the email address with yours. 

 

How do I make the login page / sign in page / calendar page / news / store / blog  to work? 

Please remember that this is a site template and is not connected to any CMS or any databases, so these features are mearly template designs, they are HTML and CSS and need to be connected to a back-end to work. 

 

 What is the Mobile UI section?

The mobile UI section is comprised of elements that you can use to create an app. Please note that converting the item to an app is entirely up to you. If these elements don't suit you and you wish to reduce the page load, simply open framework.css and scroll to the very bottom. You can remove them from there. 

This uses AJAX? How to I use a simple link? They don't work! 

Any link that points outside the main item should have the class default-link. This will terminate the smooth state / ajax page loading for that link and it will work like anyother regular link! 

 

I don't want AJAX. I want the old school loading style. How? 

That's actually very easy. Open custom.js and delete lines 849 to 890 

 

I'm a tech guru. I want more API's for the AJAX. I need them. 

Everything you need to know to customize the AJAX and more API's can be located on the plugin page over at https://github.com/miguel-perez/smoothState.js

 

I want to build an app using PhoneGap and Cordova. Can you help me? 

We only provide support for the item as is, meaning a site template. PhoneGap and Cordova's pages state clearly that they work wiht ANY html, css and JS item on the planet. So compatiiblity is not an issue. Furthermore, we use the latest and easiest code techniques to assure there won't be issues. 

 

Do I need an extended license?

If your app will be paid or have in-app paid features, yes. You do. We will pursue legal action against any app on any application store that uses our code and can't provide a license for an extended license use.

Let's explain everything in basic human words, so you don't get headaches trying to understand this. It's quite VERY simple! 

The header is comprised of 5 main elements. 4 icons and the logo. Each element is placed using the first class, header-icon-NUMBER while the second class dictates the activation for that element. For example, heading-icon-1 in our case will deploy the left sidebar, but, if you want to use it as a link, you just remove the secondary class and fill in the href. 

Replacing the logo is simple as well. You can locate the header-logo class inside menu.css on line 121 and line 131, depending on the color you choose for your header. Replace the image there with yours and make sure to change the widht, height and background-size parameteres to half of your image to maintain aspect ratio.

Sidebar Menu

First, we'll explain how to edit the sidebar header, dividers, and menu items, then, for greater ease of understanding, we'll explain the color shift.

Creating a Menu Item

To create a simple menu item, follow the structure below

                <a class="menu-item" href="page-sitemap.html">
                    <i class="fa fa-sitemap bg-night-dark"></i>
                    <em>Sitemap</em>
                    <i class="fa fa-circle"></i>
                </a>

Line 1 is the menu housing box 

Line 2 is the icon, and the background of the icon. The available colors are presented at the bottom of this section

Line 3 is the text 

Line 4 is the circle to the right 

Line 5 closes the menu. 

To make this item active, simply add the secondary class menu-item-active to line 1

Creating a Menu with Submenus

                <div class="has-submenu">
                    <a class="menu-item show-submenu" href="#">
                        <i class="fa fa-newspaper-o bg-orange-dark"></i>
                        <em>News</em>
                        <strong>3</strong>
                    </a> 

                    <div class="submenu">
                        <a class="submenu-item" href="#">      <i class="fa fa-angle-right"></i><em>      News Page          </em><i class="fa fa-circle"></i></a>
                        <a class="submenu-item" href="#">   <i class="fa fa-angle-right"></i><em>      News Article       </em><i class="fa fa-circle"></i></a>
                        <a class="submenu-item" #">     <i class="fa fa-angle-right"></i><em>      News Article List  </em><i class="fa fa-circle"></i></a>
                    </div> 
                </div>

Line 1 is the main class called has-submenu to house all the menu that will follow

Line 2,3,4,5,6 are the same elements as a regular menu item

Lines 8,9,10,11,12 are the submenu, inside here there are anchors with text and icons. 

Line 13 closes the main menu.

To make the submenu selected and activated, on line 2 add the third class submenu-active, on line 8 add the secondary class submenu-active and on the submenu anchor you want, add the class submenu-item-active.

That's it! That's all there is to the sidebar structure

Sidebar Available Colors 

Sidebars are available in a few color choices, dark and light + background or no background for icons. To change this simply replace the secondary class of sidebar-scroll-left and sidebar-scroll-right with sidebar-dark-clean, sidebar-light-clean, sidebar-light or sidebar-dark.

The material menu can house up to 6 icons. This is done because of the fact that mobile devices have a limited available height, and although you may be rocking the biggest Note or Iphone out there chances are some of your users still have smaller devices. Further customizing of this is up to you at your own risk! 

The material menu houses anchors that represent the circles. Each have 2 classes, mm and material-menu-NUMBER (up to 6) 

You can choose the position of the menu by using the class material-menu-left or material-menu-right and change the background and color of it by replacing material-menu-dark with material-menu-light and material-dark-background with material-light-background. 

 

 

 

 

Mobile Homescreen

You must add one of the following classes in order to generate the icon style. round-icons, circle-icons, square-icons. Only one of the last 3 classes must be added. 

Main Icons at the Bottom

The main icons at the bottom are generated by the following code. 

    <div class="home-main-icons">
        <div class="home-slide-icons four-rows four-columns">

            <a href="index.html" class="icon-1"><i class="bg-red-dark fa fa-home"></i><em>Home</em></a>
            <a href="page-sitemap.html" class="icon-2"><i class="bg-green-dark fa fa-cog"></i><em>Features</em></a>
            <a href="gallery-round.html" class="icon-3"><i class="bg-blue-dark fa fa-camera"></i><em>Galleries</em></a>
            <a href="portfolio-one.html" class="icon-4"><i class="bg-orange-dark fa fa-image"></i><em>Portfolios</em></a>                        
        
        </div>
    </div>

The actual icons are housed insde two classes. One is use to animate them when swiping, the other is used to generate their numbers. Replace four-rows and four-columns with either three-rows and three-columns or five-rows and five-columns. 

Be sure to use the right number of icons inside once you select a new value. For example, if you select 3 rows and 3 columns you can use 3 icons. 

Each icon anchor class, you will find the font-awesome icon required to generate the icon, and classes for the background color. Use the background colors exaplined in the feature documentation. We have a few available! 

Generating the Large Icon Layout

The layout for the icons is added inside a slider, show below is it's strucutre

<div class="homepage-cover-slider">
    <div class="swiper-wrapper">
    <div class="cover-screen">
        <div class="home-intro">
        <!-- Main Intro Here -->
        </div>
    </div>
    </div>
    <div class="swiper-wrapper">
    <div class="cover-screen">
        <!-- Icons Here --> 
    </div>
    </div>
    <div class="swiper-wrapper">
    <div class="cover-screen">
        <div class="home-outro">
        <!-- Outro Here -->
        </div>
    </div>
    </div>
</div>

Inside the cover slider, you will find cover slides. At the moment, the first and last are text. Be sure you use home-intro and home-outro classes to house the texts in them. You can add more text or content, just be sure to center it to look nicely! 

Inside the second cover screen, you'll add the icons. This is how. 

            <div class="home-slide-icons four-rows four-columns">
                <a href="index.html"><i class="bg-red-dark fa fa-home"></i><em>Home</em></a>
            </div>

We used just 1 icon in it, but the way you use it is similar to the mai icons, but with a catch! If you use four-rows, four-columns, then you must have 4x4 icons, that's 16. If you use 3 rows and 3 columns, then 9 icon, five rows five columns? 25 icons! 

That's all on this part! Now, let's get to animating the bottom part! 

In custom.js on line 138 to 146 you'll find the next piece of code 

        function check_class(){
            if(swiper_homecover.activeIndex == 1 ){
                $('.home-main-icons').addClass('show-main-icons');  
                $('.header').addClass('hide-header-icons');
            } else {
                $('.home-main-icons').removeClass('show-main-icons');
                $('.header').removeClass('hide-header-icons');
            }
        }

This code will tell the slider that if it's on page 2 ( because 1 = 2 because counting starts from 0 ) to hide the header and show the icons at the bottom.
